# 社交阅读与通知中心设计

## 背景

当前项目已经具备多用户博客的核心能力：注册登录、用户资料、文章发布、草稿、公开/私密可见性、评论回复、好友申请、私信、归档、标签和 RSS。

下一轮完善应优先补齐“多用户之间如何发现作者、保存文章、接收互动反馈”。这比继续追加零散按钮更有价值，因为现有评论、好友、私信功能已经产生了用户间互动，但缺少一个统一的作者入口、收藏入口和通知入口。

本设计保持现有 Django 函数视图、模板和单个 `blog` 应用结构，不引入新的前端框架，不大规模重排页面，不改变已有文章可见性规则。

## 目标

- 新增作者主页，让用户可以查看某个作者的公开资料和可访问文章。
- 新增文章收藏，让登录用户保存可访问文章，并在“我的收藏”页面集中查看。
- 新增通知中心，让评论、回复、好友申请、私信等互动产生可追踪的未读通知。
- 将首页模板改为继承 `base.html`，减少导航栏、页脚和基础样式重复。
- 为新增权限、可见性和通知触发逻辑补充测试。

## 非目标

- 不实现实时 WebSocket 聊天或实时通知推送。
- 不实现点赞、文章评分或复杂推荐算法。
- 不把 `Post.tags` 迁移为独立标签模型。
- 不重构整个 `views.py` 或拆分多个 Django app。
- 不升级 Django 版本。
- 不修改生产部署结构。

## 数据模型

### PostFavorite

新增 `PostFavorite` 模型表示用户收藏文章。

字段：

- `user`：收藏者，关联 `User`。
- `post`：被收藏的文章，关联 `Post`。
- `created_at`：收藏时间。

约束：

- `user + post` 唯一，避免重复收藏。
- 按 `created_at` 倒序展示。

收藏权限：

- 用户只能收藏自己当前可访问的文章。
- 游客不能收藏。
- 作者收藏自己的文章允许，不额外禁止。
- 如果文章后来变为私密且收藏者不再有权限，“我的收藏”页面不展示该文章。

### Notification

新增 `Notification` 模型表示站内通知。

字段：

- `recipient`：接收通知的用户。
- `actor`：触发通知的用户，可为空。
- `notification_type`：通知类型。
- `post`：相关文章，可为空。
- `comment`：相关评论或回复，可为空。
- `private_message`：相关私信，可为空。
- `friend_request`：相关好友申请，可为空。
- `message`：给用户看的短文本。
- `target_url`：点击通知后的跳转地址。
- `is_read`：是否已读。
- `created_at`：通知生成时间。

通知类型第一版包含：

- `comment_on_post`：有人评论了你的公开文章。
- `reply_to_comment`：有人回复了你的评论。
- `friend_request_received`：有人向你发送好友申请。
- `friend_request_accepted`：对方接受了你的好友申请。
- `private_message`：好友给你发送了私信。

通知生成规则：

- 自己触发给自己的行为不生成通知。例如作者评论自己的文章时，不通知自己。
- 私信仍保留现有未读私信计数；通知中心只是额外提供统一入口。
- 删除文章、评论或私信后，通知可以保留，但相关外键使用 `SET_NULL`，避免历史通知导致对象删除失败。
- `target_url` 在通知生成时保存，后续即使关联对象被删除，通知列表仍能显示文本；点击已失效链接时由原页面返回 404 或重定向。

## 路由

新增路由：

- `/users/<str:username>/`：作者主页，命名为 `author_profile`。
- `/favorites/`：我的收藏，命名为 `favorite_posts`。
- `/post/<int:post_id>/favorite/`：收藏或取消收藏文章，命名为 `toggle_favorite`，只接受 POST。
- `/notifications/`：通知中心，命名为 `notifications`。
- `/notifications/<int:notification_id>/read/`：单条通知标记已读并跳转，命名为 `read_notification`，只接受 POST。
- `/notifications/read_all/`：全部标记已读，命名为 `mark_all_notifications_read`，只接受 POST。

已有路由保持不变。

## 作者主页

作者主页展示：

- 头像、昵称、用户名和个人简介。
- 公开资料中的 GitHub、微博链接。
- 该作者文章统计：可访问的已发布文章数、总浏览量。
- 该作者的可访问已发布文章列表，按发布时间倒序分页。

可见性规则复用现有首页逻辑：

- 游客只能看到 `published + public` 文章。
- 登录用户可以看到 `published + public` 文章，以及自己作为作者的已发布私密文章。
- 草稿不展示。

入口调整：

- 首页文章卡片作者链接跳转到作者主页。
- 文章详情页作者信息跳转到作者主页。
- 归档页作者名称可以跳转到作者主页。

## 文章收藏

文章详情页新增收藏按钮：

- 未收藏时显示“收藏”。
- 已收藏时显示“已收藏”或“取消收藏”。
- 按钮使用 POST 表单和 CSRF。
- 游客看到登录提示，不展示可提交收藏表单。

收藏行为：

- 第一次 POST 创建收藏。
- 已收藏时再次 POST 取消收藏。
- 成功后回到来源页；默认回到文章详情页。
- 如果文章不可访问，返回 404。

“我的收藏”页面：

- 只允许登录用户访问。
- 展示当前用户收藏且当前仍可访问的文章。
- 支持分页。
- 每条展示标题、作者、分类、发布时间、收藏时间和封面缩略图。
- 空状态提示用户去首页浏览文章。

## 通知中心

导航栏新增“通知”入口：

- 登录用户可见。
- 显示未读通知数量徽标。
- 保留现有好友申请和未读私信徽标，不移除已有入口。

通知列表：

- 按创建时间倒序。
- 未读通知有明显背景。
- 每条显示触发用户、通知文案、时间和操作按钮。
- 单条“查看”按钮使用 POST 标记已读后跳转到 `target_url`。
- 顶部提供“全部标记已读”按钮。

触发点：

- `add_comment`：评论文章时通知文章作者；回复评论时通知被回复评论作者，并在必要时通知文章作者。
- `send_friend_request`：发送申请时通知接收者。
- `accept_friend_request`：接受申请时通知申请人。
- `conversation_view`：发送私信时通知接收者。

避免重复和噪音：

- 回复自己不通知自己。
- 如果回复发生在自己的文章下，且被回复者和文章作者是同一个人，只生成一条通知。
- 私信每条消息生成一条通知，第一版不合并会话通知。

## 首页模板结构整理

当前 `index.html` 是完整 HTML 页面，重复了 `base.html` 的导航栏、页脚、静态资源和基础样式。

第一版整理目标：

- 将 `index.html` 改为 `{% extends 'base.html' %}`。
- 保留首页特有的 hero、搜索、卡片、侧栏和脚本。
- 把首页专属 CSS 放入 `extra_css`。
- 把首页专属 JS 放入 `extra_js`。
- 不重写页面视觉风格。
- 不改动第三方静态资源目录。

这一步降低后续新增通知入口、收藏入口时的重复修改风险。

## 权限与安全

- 所有写操作继续使用 POST 和 CSRF。
- 收藏、通知已读、全部已读都必须登录。
- 通知只能由接收者本人查看或标记已读。
- 作者主页不得泄露他人的私密文章或草稿。
- “我的收藏”不得泄露后来被改为私密且当前用户无权访问的文章。
- 私信通知不能绕过现有好友校验；发送私信仍必须是好友关系。

## 后台管理

Django Admin 新增：

- `PostFavoriteAdmin`：按用户、文章、收藏时间搜索和筛选。
- `NotificationAdmin`：按接收者、类型、已读状态、创建时间搜索和筛选。

后台只作为排查工具，不作为主要用户界面。

## 实施顺序

1. 新增模型、迁移和 Admin 配置。
2. 增加可访问文章查询 helper，供作者主页、收藏和文章详情复用。
3. 实现作者主页路由、视图、模板和入口链接。
4. 实现收藏切换、我的收藏页面和文章详情收藏按钮。
5. 实现通知模型 helper、触发点、通知列表和导航栏计数。
6. 整理首页模板继承 `base.html`。
7. 补充测试并运行检查。

## 测试计划

新增回归测试覆盖：

- 作者主页对游客只展示公开已发布文章。
- 作者本人能在自己的作者主页看到自己的已发布私密文章。
- 作者主页不展示草稿。
- 登录用户可以收藏可访问文章。
- 游客不能收藏文章。
- 用户不能收藏自己无权访问的私密文章。
- 重复收藏 POST 会取消收藏。
- “我的收藏”只展示当前仍可访问的收藏文章。
- 评论文章会通知文章作者。
- 回复评论会通知被回复用户。
- 自己触发给自己的行为不生成通知。
- 发送好友申请会通知接收者。
- 接受好友申请会通知申请人。
- 发送私信会通知接收者。
- 通知列表只显示当前用户自己的通知。
- 单条通知标记已读后跳转到目标地址。
- 全部标记已读只影响当前用户。
- 导航栏显示未读通知数量。
- 首页改为继承 `base.html` 后仍包含归档、标签、登录用户菜单、搜索表单和文章列表。

## 发布与兼容

- 新增迁移必须同时兼容 SQLite 和 MySQL。
- 线上部署仍使用现有 `sudo clover-blog-deploy`。
- 新增通知和收藏表为空时不影响已有文章、评论、好友和私信数据。
- 如果部署后通知入口显示为 0，不影响既有私信和好友申请徽标。

## 设计取舍

本设计选择“作者主页 + 收藏 + 通知中心 + 首页结构整理”作为一个产品化切片，而不是一次性加入更多功能。原因是这四部分共享同一条用户路径：读文章、认识作者、保存文章、收到互动反馈。它们能明显补齐当前多用户博客体验，同时范围仍然可测试、可迁移、可部署。

实时推送、点赞、推荐、全文搜索和复杂编辑器都可以作为后续迭代，但不进入本轮，避免第一轮改动过大。
