# 文章与草稿删除改为 POST

## 目标

文章和草稿删除操作只接受 POST 请求，防止普通链接访问、浏览器预加载或爬虫访问触发数据删除。

## 后端设计

- `delete_post` 和 `delete_draft` 继续要求用户登录。
- 两个视图增加 `require_POST` 限制。
- GET、HEAD 等非 POST 请求由 Django 返回 `405 Method Not Allowed`，且不删除数据。
- 保留现有作者权限校验和文章状态校验。
- 不修改路由地址、数据模型或数据库迁移。

## 前端设计

- 草稿箱的删除链接改为 POST 表单。
- 文章详情页的删除链接改为 POST 表单。
- 两个表单都包含 Django CSRF 令牌。
- 删除按钮继续显示原有确认弹窗；用户取消时不提交表单。
- 按钮外观和文案保持不变。

## 测试

- 验证草稿作者可以通过 POST 删除自己的草稿。
- 验证 GET 不能删除草稿，并返回 405。
- 验证已发布文章作者可以通过 POST 删除自己的文章。
- 验证 GET 不能删除已发布文章，并返回 405。
- 运行 Django 系统检查和 `blog` 完整测试集。
